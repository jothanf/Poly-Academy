Vistas Base de PolyAcademy

Descripción General

Las vistas base de PolyAcademy proporcionan la estructura fundamental para gestionar las interacciones entre usuarios y el sistema. Estas vistas están diseñadas para satisfacer las necesidades tanto de estudiantes como de profesores.

BaseListView

Como usuario, necesito ver listas de elementos (cursos, clases, etc.) de manera organizada y paginada.

  • Paginación automática de resultados
  • Filtrado por parámetros de búsqueda
  • Ordenamiento personalizable
  • Caché de resultados para mejor rendimiento

BaseDetailView

Como usuario, necesito ver los detalles completos de un elemento específico.

  • Recuperación de objeto individual
  • Validación de permisos de acceso
  • Inclusión de relaciones relevantes
  • Manejo de objetos no encontrados

BaseCreateView

Como usuario autorizado, necesito crear nuevos elementos en el sistema.

  • Validación de datos de entrada
  • Asignación automática de usuario creador
  • Manejo de archivos multimedia
  • Respuestas personalizadas de éxito/error

BaseUpdateView

Como usuario autorizado, necesito modificar elementos existentes.

  • Actualización parcial o completa
  • Validación de permisos de edición
  • Registro de cambios históricos
  • Manejo de conflictos de concurrencia

BaseDeleteView

Como usuario autorizado, necesito eliminar elementos del sistema.

  • Eliminación segura con confirmación
  • Validación de dependencias
  • Eliminación en cascada configurable
  • Papelera de reciclaje temporal

BaseAuthView

Como usuario, necesito autenticarme de forma segura en el sistema.

  • Gestión de tokens JWT
  • Validación de credenciales
  • Control de sesiones activas
  • Renovación automática de tokens

BasePermissionView

Como administrador, necesito gestionar los permisos de acceso.

  • Control granular de permisos
  • Roles predefinidos y personalizados
  • Herencia de permisos
  • Registro de accesos y cambios

BaseErrorHandler

Como desarrollador, necesito manejar errores de forma consistente.

  • Respuestas de error estandarizadas
  • Logging detallado de errores
  • Notificaciones de error configurables
  • Recuperación automática cuando sea posible

Beneficios Clave

  • Consistencia en la interfaz de API
  • Reutilización de código común
  • Manejo estandarizado de errores
  • Seguridad integrada
  • Facilidad de mantenimiento
