Vistas de Contenido de Clase - Historias de Usuario

Listar Contenido (GET)

Como profesor o estudiante, necesito ver una lista ordenada de todo el contenido de una clase específica.

  • Puedo filtrar el contenido por class_id
  • El contenido se muestra ordenado según el campo 'order'
  • Recibo una respuesta estructurada con status y mensaje
  • En caso de error, recibo detalles específicos del problema

Crear Contenido (POST)

Como profesor, necesito crear nuevo contenido para una clase con diversos tipos de media.

  • Puedo subir múltiples archivos multimedia
  • El sistema valida que el content_details sea JSON válido
  • Recibo confirmación de creación exitosa
  • En caso de error de validación, recibo detalles específicos
  • Se maneja la carga de archivos de forma segura

Actualizar Contenido (PUT)

Como profesor, necesito actualizar el contenido existente de una clase.

  • Puedo modificar todos los campos del contenido
  • El sistema valida los nuevos datos
  • Recibo confirmación de actualización exitosa
  • Se mantiene la integridad de los archivos multimedia
  • En caso de error, recibo información detallada

Actualización Parcial (PATCH)

Como profesor, necesito actualizar campos específicos del contenido sin afectar el resto.

  • Puedo actualizar solo los campos necesarios
  • El sistema mantiene los campos no modificados
  • Recibo confirmación de actualización parcial exitosa
  • Se validan solo los campos actualizados

Eliminar Contenido (DELETE)

Como profesor, necesito eliminar contenido de clase de forma segura.

  • Se eliminan los archivos multimedia asociados
  • Recibo confirmación de eliminación exitosa
  • Se mantiene la integridad referencial de la base de datos
  • En caso de error, se evita la eliminación parcial

Manejo de Errores

Como usuario del sistema, necesito recibir información clara sobre errores.

  • Mensajes de error descriptivos y amigables
  • Códigos de estado HTTP apropiados
  • Detalles específicos de validación cuando aplica
  • Registro de errores para depuración

Características Técnicas

  • Hereda de BaseModelViewSet para funcionalidad común
  • Soporta múltiples formatos de datos (MultiPart, Form, JSON)
  • Manejo robusto de archivos multimedia
  • Validación de JSON en content_details
  • Respuestas estructuradas consistentes
