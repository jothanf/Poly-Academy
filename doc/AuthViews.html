    Documentación de Auth Views

    Descripción General

    Las vistas de autenticación proporcionan la funcionalidad necesaria para gestionar el acceso seguro de usuarios a la plataforma. Estas vistas permiten un sistema de autenticación unificado para estudiantes y profesores, asegurando que solo los usuarios autorizados puedan acceder a los recursos protegidos.

    Inicio de Sesión Unificado

    Como usuario, necesito poder iniciar sesión en la plataforma de manera segura.

    • Autenticación por email y contraseña
    • Validación de credenciales
    • Identificación automática del rol
    • Generación de tokens JWT

    Cierre de Sesión

    Como usuario, necesito poder cerrar mi sesión de forma segura.

    • Invalidación de tokens activos
    • Blacklist de refresh tokens
    • Confirmación de cierre exitoso
    • Manejo de errores en el proceso

    Validación de Roles

    Como sistema, necesito validar el tipo de usuario que intenta acceder.

    • Verificación de rol estudiante
    • Verificación de rol profesor
    • Prevención de roles duplicados
    • Manejo de usuarios sin rol

    Gestión de Errores

    Como sistema, necesito manejar los errores de autenticación de forma clara.

    • Validación de campos requeridos
    • Mensajes de error personalizados
    • Códigos de estado HTTP apropiados
    • Logging de errores del sistema

    Características Técnicas

    Detalles técnicos importantes de la implementación

    • Autenticación basada en JWT
    • Documentación con drf-spectacular
    • Respuestas estandarizadas
    • Manejo de permisos DRF

    Beneficios Clave

    • Sistema de autenticación unificado
    • Gestión segura de sesiones
    • Control granular de roles
    • Experiencia de usuario mejorada
    • Manejo robusto de errores

    Unificado Login

    Permite a los estudiantes y profesores iniciar sesión en la plataforma.

    • Endpoint: /api/auth/unified_login
    • Método: POST
    • Parámetros:
      • email: Email del usuario.
      • password: Contraseña del usuario.
    • Respuesta:
      • 200: Inicio de sesión exitoso con token de acceso.
      • 400: Error de validación (email o contraseña faltantes).
      • 401: Contraseña incorrecta.
      • 403: Usuario no tiene un rol asignado.
      • 404: Usuario no encontrado.
      • 500: Error interno del servidor.

    Unificado Logout

    Cierra la sesión del usuario invalidando su token.

    • Endpoint: /api/auth/unified_logout
    • Método: POST
    • Respuesta:
      • 200: Sesión cerrada exitosamente.
      • 500: Error interno del servidor.
